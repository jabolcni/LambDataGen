üìñ Overview

Lamb Distributed Runner is a client-server system for distributed generation of chess training data using the Lambergar (executable is naed lamb) engine. Multiple computers work together generating chess positions and games, with centralized monitoring and control.

üöÄ Quick Start

1. Start the Server

python server.py

Server runs on http://0.0.0.0:5001

2. Run Clients on Worker Machines

python client.py --name worker1 --concurrency 4 --server http://SERVER_IP:5001

3. Monitor via Web GUI

Access http://SERVER_IP:5001 to view live status and control the system.

üí° Tip: Start with 10 games to verify setup, then increase to 100-1000 games for production runs.

üèóÔ∏è Architecture

Server (server.py)

  • Flask web server with REST API and web GUI
  • Central coordination for multiple clients
  • Real-time monitoring of progress and statistics
  • File storage for generated .bin files
  • SQLite database for tracking client progress

Client (client.py)

  • Multi-process worker that runs lamb datagen commands
  • Automatic registration with the server
  • Progress reporting and file upload
  • Self-managing - automatically uses new parameters
  • Disk space management - cleans up old files

üêõ Troubleshooting

Common Issues & Solutions

‚ùå Problem: Client can't connect to server
‚úÖ Solution: Check firewall settings, verify server IP and port, ensure port forwarding
‚ùå Problem: lamb binary not found
‚úÖ Solution: Ensure ./lamb exists and is executable: chmod +x ./lamb
‚ùå Problem: Multiple lamb processes accumulating
‚úÖ Solution: Kill all processes: pkill -f lamb

Performance Tips

  • Concurrency: Set to number of CPU cores
  • Games per batch: 10-100 for quick feedback, 1000+ for long runs
  • Network: Use wired Ethernet for large file uploads
  • Storage: Ensure server has enough disk space for .bin files

System Monitoring Commands

üíæ File Management

Generated Files

  • Output format: data_YYYYMMDD_HHMMSS_NAME_XXXX.bin
  • Server storage: server_data/games/ directory
  • Client cleanup: Automatic when folder exceeds 4GB (keeps 2GB)

Database Schema

CREATE TABLE clients ( client_id TEXT PRIMARY KEY, name TEXT, ip TEXT, last_seen TEXT ); CREATE TABLE runs ( id INTEGER PRIMARY KEY AUTOINCREMENT, client_id TEXT, output_file TEXT, games_completed INTEGER DEFAULT 0, positions_completed INTEGER DEFAULT 0, status TEXT, timestamp TEXT, FOREIGN KEY(client_id) REFERENCES clients(client_id) );

File Sizes

  • Typical size: 40-100MB per 100 games
  • Binary format compatible with lamb engine
  • Automatically managed to prevent disk exhaustion
